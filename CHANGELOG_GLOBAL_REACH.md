# TCGPlay 2.0 改造交付说明

本次改造基于你现有的 **Weioiiaoiia/tcgplay2.0** 代码库直接进行，没有新建项目。整体工作围绕两个目标展开：一是将站点融合为可运行的四语言界面；二是按你的要求清理不需要的模块、虚假展示与不符合气质的文案表达。

| 模块 | 处理结果 | 说明 |
| --- | --- | --- |
| 多语言生态 | 已接入 | 支持简体中文、繁體中文、English、日本語、한국어 五个语言代码映射，其中实际覆盖四类语言生态，并支持繁简中文分别显示 |
| 右上角语言切换器 | 已新增 | 集成到导航右上角，支持即时切换，无需刷新页面 |
| 浏览器语言自动检测 | 已接入 | 首次进入优先读取浏览器语言，并持久化到本地存储 |
| 卡库菜单表情 | 已删除 | 统一改为纯文字，风格更简洁 |
| 两个功能介绍卡片 | 已删除 | 已删除“智能钱包”“多语言生态”两张功能卡，仅保留 9 个功能 |
| 协议状态条 | 已删除 | 已删除你截图中页脚上方的协议状态模块 |
| 虚假统计数字 | 已删除 | 已删除首页首屏中的三项统计数字展示 |
| 文案风格 | 已统一 | 清理土味、夸张、玄乎表述，改为克制、专业、国际化语气 |

本次国际化采用运行时切换方案，核心是 **i18next + react-i18next + 浏览器语言检测**。这样做的好处是无需刷新页面即可全局更新文案，同时适合当前仓库的 React/Vite 架构，也便于你后续继续扩展更多页面或更多语种。

## 关键实现说明

右上角语言切换器已经作为独立组件接入导航栏。桌面端使用下拉式切换，移动或紧凑区域可以使用简化按钮形式。切换时通过动画过渡实现轻微的淡入与缩放，整体视觉保持暗色、克制、偏高端的现有风格。

国际化资源集中整理到了统一资源文件中，便于后续维护。首页首屏、导航、功能区、市场图鉴、卡牌故事、卡牌详情、我的卡库、钱包连接、404 页面等核心模块均已改为依赖翻译资源而不是写死文案。卡牌名称、描述、属性和流转记录也已经做成可按语言切换的本地化数据输出。

针对你的文案要求，我已经重点清理了容易显得土味、玄幻、过度修饰的表达。像“女神”“苍穹”这类风格不统一的措辞已经避免继续出现在卡牌标题、描述、故事语句和功能文案中，整体改成更像国际化收藏平台会使用的专业表达。

当前仍有一项非阻塞性提示：`index.html` 中的分析脚本占位变量 `VITE_ANALYTICS_ENDPOINT` 与 `VITE_ANALYTICS_WEBSITE_ID` 未配置，因此构建时会出现警告。这不是本次改造引入的问题，也不会阻止项目构建成功；如果你要上线生产环境，后续可以再补充这两个环境变量或暂时移除对应脚本。

## 你接下来可以怎么用

你现在可以直接在本地运行项目进行验收，重点查看以下几点：导航右上角语言切换是否即时生效；首页、功能区、市场图鉴、卡库、详情页是否随语言一起变化；卡库菜单是否已变成纯文字；九项功能是否已正确保留；虚假统计数字和协议状态条是否已经完全消失。

如果你愿意，我下一步还可以继续帮你做两件事：第一，把剩余零散页面和未来新增内容进一步抽成更规范的翻译键体系；第二，直接帮你整理成一个可提交的 Git 提交，甚至继续推送到你仓库的新分支。
